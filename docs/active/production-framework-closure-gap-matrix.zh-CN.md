## 当前判断

如果把理想状态定义为“OPL 能生产级托管 MAS/MAG/RCA 的长时间 stage attempt，同时 direct skill path 等价、domain truth 不迁出、旧默认面退役干净”，当前约为 `65-75% framework/control-plane landed`，仍有 `25-35% production closure remaining`。

这个比例是工程判断，不是测试分数。已落地部分主要是 framework 骨架和控制面：shared contracts、domain descriptors、Temporal provider code、本机 service lifecycle、worker lifecycle contract、repo-native Temporal live proof、typed family queue、stage attempt ledger、Codex runner repo/test harness、Agent Executor Adapter、Aion stage-attempt workbench projection，以及 MAS/MAG/RCA descriptor alignment。

未闭合部分集中在 production closure 和受证据门槛约束的 functional follow-through：production Temporal readiness / residency、真实长时 domain activity soak、MAS/MAG/RCA domain owner receipt chain、domain memory body / writeback apply、统一 domain-agent descriptor CLI、domain repo physical skeleton layout，以及 active-path legacy residue 的最终物理退役。

2026-05-13 补充：真实 paper/grant/visual 长时 soak 属于 production evidence gate；排除这类耗时运行后，核心功能性 surface 已按 `docs/active/production-functional-closure-plan.zh-CN.md` 推进并合入 main。当前不能写成“所有功能性要求都已完成”：仍待闭合的是当前本机 production provider readiness、真实 domain memory body / writeback apply、破坏性 physical skeleton path move、legacy physical delete、统一 descriptor CLI 和长时 SLO 校准。

## 统一口径

当前可以写成“OPL 已经具备完整生产级智能体框架的控制面骨架”，不能写成“OPL 已经全量生产可用”，也不能写成“除了长时测试以外所有功能性要求都完成”。准确边界是：framework contracts、runtime provider code、attempt ledger、executor adapter、domain descriptors、receipt/ref surfaces 和 workbench projection 已落地；production readiness/residency、真实 domain owner receipt chain、真实 memory body / writeback apply、统一 descriptor CLI、physical skeleton path migration 和 legacy physical retirement 仍是未完成闭环。

后续实现不需要再新建平行大计划。下一跳应直接围绕 `Temporal production residency -> MAS paper-line guarded apply soak -> domain memory/lifecycle apply generalization -> MAG/RCA controlled soak -> physical skeleton layout -> legacy physical retirement` 收口，每一步都必须留下 repo-native verification、domain owner receipt 或 typed blocker。
