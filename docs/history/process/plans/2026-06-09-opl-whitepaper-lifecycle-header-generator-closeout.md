Owner: `One Person Lab`

State: `history_provenance`
Machine boundary: 本文只记录 OPL public whitepaper lifecycle-header generator 修复的覆盖、SSOT 决策、验证边界和未声明范围。当前机器真相继续归 `contracts/`、source、CLI/API 行为、runtime ledger、provider receipt、domain-owned manifests、App evidence、generated whitepaper source JSON 和真实 workspace evidence；本文不得作为 readiness claim、owner receipt、typed blocker、quality/export verdict、artifact authority、physical delete authorization 或 compatibility surface。

## Scope

本轮治理主题是 `docs/public/whitepaper/opl-whitepaper.md` lifecycle header 缺口。OPL Doc doctor 已能识别核心 active truth，但仍提示 generated Markdown 白皮书缺少 `Owner` / `Purpose` / `State` / `Machine boundary`。

白皮书目录自身声明 source-first 规则：`docs/public/whitepaper/opl-whitepaper.source.json` 是 canonical 内容源，Markdown、PDF 和 verification JSON 都是生成物。因此本轮不手改 generated Markdown 正文，而是修 `scripts/build-opl-whitepaper.ts`，让 source JSON 已有的 lifecycle metadata 进入生成 Markdown，再重新运行 `npm run docs:whitepaper`。

## SSOT Decisions

| Semantic theme | Single Source of Truth |
| --- | --- |
| Whitepaper content, owner, purpose, state, machine boundary | `docs/public/whitepaper/opl-whitepaper.source.json` |
| Whitepaper Markdown / PDF / verification JSON | Generated by `scripts/build-opl-whitepaper.ts` via `npm run docs:whitepaper` |
| Public whitepaper directory lifecycle | `docs/public/whitepaper/README.md` |
| OPL current product/runtime truth | Core docs, contracts, source, CLI/read-model, runtime ledger, provider receipts, domain manifests and App evidence |

## Retired / Guarded Surfaces

The retired practice is direct/manual lifecycle-header edits to generated `opl-whitepaper.md`. Future whitepaper lifecycle or body changes must update `opl-whitepaper.source.json` or `scripts/build-opl-whitepaper.ts`, then run `npm run docs:whitepaper`.

This does not close OPL series docs-governance globally, does not certify App release readiness, production readiness, domain readiness, owner receipts, typed blockers, artifact authority or full `README*` / `docs/**/*.md` paragraph coverage.
