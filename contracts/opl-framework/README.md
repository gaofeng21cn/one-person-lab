# OPL Framework Contracts

Owner: `One Person Lab`
Purpose: `opl_framework_contract_support_index`
State: `active_support`
Machine boundary: This document is a human-readable support index for OPL Framework contracts. Machine truth remains in JSON contract files, source, tests, CLI/read-model output, runtime ledgers, and provider/domain receipts.

This directory preserves the active `OPL Framework` runtime and family control-plane contract corpus. `One Person Lab App` and Foundry Agents may consume these contracts, but this directory does not define a second runtime truth for the App or any domain agent.

It is repo-tracked because the current framework needs stable machine-readable inputs for:

- stage-led task selection
- product-layer ownership for `OPL Framework`, `One Person Lab App`, and `Foundry Agents`
- admitted domain-agent / Foundry package catalog projection
- provider-backed runtime attempts
- domain-neutral transition table runner and matrix evaluation
- stage graph / route-as-transition runtime support for complex domain agents where route is a domain transition recommendation and stage attempt lifecycle remains OPL-owned
- stage graph / owner-route hydration / reconciliation / attempt-ledger boundaries for complex domain agents, where stage is the OPL attempt unit and route is a domain-owner semantic rather than a small stage
- single-writer Stage transition authority boundaries, where many modules may submit transition intents or observations but only OPL folds them into Stage current pointer, StageRun terminal state, and `current_owner_delta`
- functional agent runtime harness coverage for queue, typed closeout, refs-only memory writeback, human gate, retry, dead-letter, and repair transitions
- domain pack compiler and generated interface read model that compiles admitted domain packs or standard agent repo contracts into OPL-owned CLI / MCP / Skill / product-entry / OpenAI / AI SDK / sidecar / status / workbench / harness generated-surface handoff projections
- standard domain-agent admission gates that keep candidate identity, domain truth owner, generated default entry, clean checkout currentness before domain-handler export/dispatch, provider-hosted stage admission, provider transport start, and Codex runner launch, standard pack ABI, stage artifact contract, execution model, authority boundary, owner receipt, typed blocker, and human gate false-authority requirements machine-readable without admitting any specific domain or claiming readiness
- surface-budget governance that keeps new default surfaces limited to launch safety, authority boundary, evidence/replay/audit/route-back, or repeated App/runtime consumption
- operating-loop adoption governance that maps external Codex work-loop patterns into OPL workstream/thread continuity, goal oracle, heartbeat/steering, artifact-first review, memory refs, receipt, and read-model boundaries without importing an external runtime or authority
- OPL Flow completion-audit closeout governance for thorough-landing / complete-execution claims, so plan items, lane mappings, fresh executable evidence, missing refs, and `can_claim_complete` are machine-readable before any final completion claim
- brand-module governance that keeps Charter, Atlas, Workspace, Pack, Stagecraft, Runway, Ledger, Console, Foundry Lab, and Connect at the Workspace-level structural baseline through one registry, one command-surface governance contract, one module-surface contract, native per-module CLI/read-model families, validation/doctor gates, and false-authority boundary
- source-module governance that keeps the ten Framework brand modules visible as physical `src/modules/<module_id>/` entrypoints, maps legacy root-level `src/` files to their module owner, and reports cross-module dependency direction / public-entrypoint drift; the human maintainer guide is `docs/references/source-module-boundary.md`
- owner-delta-first readiness projection that lets default App/operator surfaces answer current safe action, waiting owner, required delta / receipt / typed blocker, and readiness blocker before exposing raw refs-only audit counters
- target operating architecture governance that freezes the ideal resource model, single Stage Transition Authority, declarative Domain Pack / authority ABI, surface lanes, small reconciler loops, Atlas/Ledger telemetry, thin App Console default, and Agent Lab improvement boundary without over-specifying diagnostic detail
- target architecture contract kernels for current owner delta, stage artifact unit, owner answer, passive evidence ledger, ordinary golden path, advisory no-progress signals, guardrail tiers, progress truth, wrapper retirement, and default surface budget without moving domain truth into OPL
- cognitive computation kernel governance that lets each Stage declare generation, reflection, comparative selection, evolution, meta-review, tool affordance boundaries, knowledge, and independent quality-gate strategy refs without turning tools into a workflow script, Route into a small Stage, or domain truth into OPL-owned state
- advisory knowledge boundary governance for domain-owned Markdown memory, so Atlas / Pack / Stagecraft / Runway / Ledger / Console / Connect can carry refs, prompt-context refs, receipts, and projections without becoming route scorers, winning-path generators, memory-body owners, quality gates, or export / publication verdict owners
- generic workspace/source/artifact/memory substrate projection and App/operator workbench grouping without moving domain truth/body/verdict/authority into OPL
- framework runtime dependency location for OPL-compatible agents
- Runtime Manager readiness and state projection
- App runtime state/action CLI boundary for GUI implementations
- optional native-helper lifecycle checks

The current product model is `OPL Framework -> One Person Lab App / CLI -> Foundry Agents`. `one-person-lab` owns the framework/runtime/CLI/contracts layer, `one-person-lab-app` owns GUI product truth and release validation, `opl-aion-shell` is the current shell implementation carrier, and MAS/MAG/RCA own their domain app/runtime authority. The execution chain remains `Codex CLI first-class executor + explicit OPL activation + configured family runtime provider + family skill sync/discovery`.

## Target Architecture Contract Groups

These schema files are the machine-readable OPL target architecture surface. Their machine boundary is framework-owned shape, refs, launch / audit / fail-closed semantics, and App/operator projection only. They do not copy domain truth, artifact bodies, memory bodies, owner receipt authority, quality verdicts, production readiness, or App release authority into OPL.

### Target Operating Architecture

- `target-operating-architecture-contract.json`: top-level target architecture contract. It freezes `Agent / DomainPack / WorkspaceGroup / ProjectUnit / StageRun / StageArtifactUnit / OwnerAnswer / EvidenceRef / ReleaseCohort / ImprovementWorkOrder` as the resource vocabulary, requires `spec/status/conditions/ownerRefs/finalizers`, keeps Stage Transition Authority single-writer, keeps Domain Pack + generated surfaces + authority functions as the standard agent ABI, routes small details to advisory/audit/diagnostic/cleanup/production-evidence lanes, splits runtime work into small reconcilers, keeps Atlas/Ledger/Console/Foundry Lab boundaries refs-only, and preserves false authority flags.
- `advisory-knowledge-boundary-contract.json`: family-level boundary for domain-owned Markdown memory. It freezes the default rule that memory refs are `reference_only_prompt_context`, OPL transports body-free refs by default, missing advisory memory does not block launch, and only source/data authority, owner identity, forbidden writes, irreversible mutation, hard reviewer/publication/final export/submission claims, or owner-receipt/typed-blocker claims become hard gates. It also defines `gate_intent` and the Console/App three-column operator projection: reference suggestions, soft gaps, and hard owner gates.
- `okf-context-bundle-contract.json`: OPL-owned OKF context bundle boundary for Google OKF v0.1-compatible Markdown/YAML/crosslink directories. It is consumed by `src/okf-context-bundle.ts` and `opl okf validate|inspect|project-pack|project-repo|native-frontmatter inspect --json` as an advisory, body-free context interchange layer over Foundry Agent pack refs and memory locator refs. `project-repo` reads a domain repo pack compiler input plus optional memory descriptor and writes a single OKF bundle without touching domain truth. Native OKF frontmatter is only an opt-in advisory migration lane for stable domain-owned `agent/**/*.md`; `native-frontmatter inspect` only reports ready/advisory-gap metadata and does not mutate domain files. It is not part of the framework startup required-contract set and does not grant runtime scheduling, domain truth, memory body, owner receipt, typed blocker, quality/export verdict, artifact authority, domain-ready, App-release-ready, or production-ready authority.

### Owner Delta Kernel

- `current-owner-delta.schema.json`: compact default owner / delta / hard-gate / action payload and ordinary next-action root.
- `owner-answer.schema.json`: unified return shape for owner receipt, typed blocker, human decision, and route-back.

### Stage Artifact Unit

- `stage-run-kernel-contract.json`: refs-only StageRun spec/status/event-log/read-model substrate, observed_generation, retry budget, hold scope, execution authorization, closeout receipt binding, and domain adoption boundary.

- `stage-transition-intent.schema.json`: refs-only transition intent envelope for domain owner answers, typed blockers, human-gate decisions, provider observations, read-model/worklist observations, Agent Lab observations, and evidence observations.
- `stage-manifest.schema.json`: Stage Folder manifest shape for required roles, produced role artifacts, input/output refs, receipt/blocker refs, content hashes, current pointer, lineage, and authority boundary.
- `role-artifact-ref.schema.json`: refs-only semantic role artifact pointer with content hash and lineage; role is the interface, not file name.
- `stage-owner-receipt.schema.json`: owner-signed stage receipt shape consumed by OPL for success or route handoff, including optional StageRun / manifest / current-pointer / source-fingerprint binding refs, without granting OPL receipt authority.
- `stage-typed-blocker.schema.json`: owner-signed typed blocker shape for blocked/deferred StageRun closeout and next safe action.
- `stage-artifact-unit.schema.json`: physical output, manifest, content hashes, owner answer, current pointer, lineage, and progress truth boundary.
- `stage-artifact-progress-truth-policy.json`: family-level rule that deliverable progress requires physical output, valid manifest, owner answer, and current pointer.
- `workspace-topology-profile.schema.json`: Workspace Group -> Project Unit -> Stage Artifact Unit -> Owner Receipt / Typed Blocker profile schema with `workspace_modes=one_off|series|portfolio`, default physical `project_collection_path=projects`, canonical `projects/<project-id>` roots for RCA/MAG/OMA deliverables and MAS studies, `deliverables` / `studies` as display or legacy aliases only, `project_stage_outputs_root=artifacts/stage_outputs`, profile version/fingerprint binding, migration history, topology event provenance, and runtime-state-as-provider-backing/provenance boundary.
- `workspace-index.schema.json`: workspace-local `workspace_index.json` instance schema for the canonical topology layer, profile binding, topology events, domain display labels, shared resource roles / manifest refs, refs-only shared resource provenance record shape, indexed project roots, stage outputs root manifest refs, project lifecycle, canonical `control/opl/projections` generated refs, canonical `control/opl/reports/workspace_report.json`, root mirror compatibility refs, user inspection roots, and authority false flags materialized by `opl workspace init|ensure|adopt --apply|upgrade --apply` and checked by `opl workspace validate|doctor`.
- `agent-workspace-norm-contract.json`: executable OPL Agent workspace norm that binds `opl workspace ensure`, App `workspace_ensure`, descriptor-only MCP/Skill/OpenAI/AI SDK delegates, workspace registry writes, user inspection paths, canonical generated projection/report roots, workspace report, fleet report, project lifecycle transitions, project delete safe gate, shared resource provenance preservation, full workspace_norm projection currentness, Contract Light workspace diagnostic severity policy, runtime-state boundary, lifecycle ownership, and conformance anti-overclaim flags.
- `source-material-ingest-contract.json`: domain-neutral source material intake contract for `opl workspace source ingest`, including sha256-addressed stored file refs, source material receipt refs, Codex/domain-agent handoff refs, and false-authority guards that prevent source copying from becoming source semantics, domain truth, owner acceptance, typed blocker, or extraction quality verdict.
- `pack-bundle-contract.json`: source-parts to generated-aggregate contract for large JSON consumer surfaces. It is consumed by `src/pack-bundle.ts` and `opl pack bundle manifest|write|check --assembly <path> --json`. Editable source parts remain the source of truth; generated aggregate JSON remains a tracked consumer compatibility surface with generated metadata and `do_not_edit=true`; the bundle manifest records source entries, source digest, expected aggregate hash, generator commands, and false-authority flags. Bundle validation proves only source/aggregate consistency and cannot claim domain ready, production ready, quality verdict, artifact authority, owner receipt, typed blocker, or domain truth mutation.
- `pack-native-helper-probe-contract.json`: provider- and domain-neutral Pack probe contract for domain-owned native helpers. `opl pack native-helper probe --descriptor <path> --json` resolves the descriptor-relative helper entrypoint plus declared runtime/tool commands and returns a deterministic `resolved|missing` receipt bound to descriptor and content SHA-256. It never executes the helper, renders PDF/image assets, mutates artifacts, signs owner receipts, creates typed blockers, or authorizes quality/publication/export readiness.
- `pack-native-helper-execution-contract.json`: generic Pack/Runway Python helper carrier. `opl pack native-helper run --catalog <catalog.json> --helper <id> --request <request.json> --json` resolves a contained domain-declared Python source root, passes only declared argv, enforces a bounded process lifetime, and requires exact JSON stdout. The receipt records transport provenance but cannot authorize domain truth, visual quality, artifact mutation, export readiness, or production readiness.
- `agent-scaffold-materialization-request.schema.json` and `agent-scaffold-materialization-contract.json`: OPL Foundry Lab's physical materialization boundary for OMA-authored semantic scaffold requests. Only declared files are replaced, object merge is limited to the descriptor and capability map, path/symlink escape fails closed, and OPL computes post-write SHA-256 plus the final build receipt. OMA's input remains a candidate and validation refs remain requested checks rather than pass/readiness claims.
- `python-executor-client-contract.json`: shared Python carrier for `opl executor run`. It owns temporary request and process-group cleanup, unwraps the canonical `AgentExecutionReceipt`, and fails closed on timeout, process failure, malformed JSON, or receipt-shape drift without taking executor or domain authority.
- `pack-os-contract.json`: generic Pack OS lifecycle contract for capability-pack descriptors, install registry entries, content-addressed cache manifests, refs-only distribution bundles, refs-only lock projection, artifact lifecycle refs, and review receipt transport. It is consumed by `src/modules/pack/pack-os.ts` and `opl pack os inspect|install|registry|cache|distribute|lock|validate --json`. Domain-owned adapters provide generic descriptors; Pack OS does not retain MAS-specific conversion or smoke contracts. Locks and distribution bundles record descriptor refs, content hashes for present local files, artifact locator refs, lifecycle state refs, review receipt refs, provenance, and authority-boundary flags without storing artifact bodies, mutating domain artifacts, signing owner receipts, or authorizing quality/readiness claims.
- `runtime-environment-substrate-contract.json`: canonical OPL runtime/dependency environment substrate contract. It binds domain dependency intent, runtime locks, content-addressed layer/cache manifests, dependency prepare locks/receipts, materialization receipts, run-context handoff and consumer preflight, cleanup plans, and doctor findings to OPL-owned false-ready boundaries. It is consumed by `src/runtime-environment-substrate.ts` and `opl runtime env inspect|lock|build|prepare|materialize|verify|doctor|run-context|cache status|cache inventory|cache prune --json` as deterministic lock / bundle projection, explicit OPL-managed materialization, verification, dependency prepare, fail-closed run-context readback, filesystem cache inventory, and protected prune receipt boundary. `materialize --apply` writes only `${OPL_STATE_DIR}/runtime-environment`; `prepare --apply` may install missing language packages only into the OPL-managed library path; `run-context --artifact-root <path>` fail-closes when refs are missing or target identity mismatches and forbids host-environment fallback; `cache prune --apply` deletes only unprotected receipt-backed runtime roots. These commands do not write development checkouts, mutate domain truth, sign owner receipts, create typed blockers, replace visual audit, or authorize App/domain/provider/publication/production readiness.
- `operator-compact-readback-contract.json`: compact operator readback contract for `opl framework readiness --family-defaults --detail compact --json` and `opl framework operating-maturity --family-defaults --detail compact --json`. These surfaces are derived from the same source surfaces as the full framework readbacks, keep default full payloads unchanged, and retain only owner/topline/false-ready fields needed for fresh gate and active-gap triage. They reduce transfer and operator-scanning payload size, but do not by themselves prove lower underlying projection compute cost. They are not second truth sources and cannot claim domain ready, App release ready, Brand L5, production ready, physical delete authority, owner receipt, typed blocker, human-gate decision, artifact readiness, or quality/export verdicts.

### Evidence Ledger

- `evidence-ledger-event.schema.json`: passive audit-only event envelope for raw evidence, provider trace, replay, receipt ledger, typed blocker groups, soak, no-regression, cleanup, and diagnostic refs.
- `substrate-provenance-surface-contract.json`: refs-only Ledger surface for project-local artifact graph refs, ledger pointer/hash, claim-warning descriptors, annotation-to-source regeneration receipt refs, and native viewer watch-only descriptors. It records OpenScience-inspired local-first provenance patterns without importing OpenScience runtime, reading or storing artifact bodies, mutating artifacts, writing domain truth, creating owner receipts, or authorizing quality/readiness claims.
- `observability-semantic-conventions-contract.json`: shared OPL Ledger / Runway / Console semantic vocabulary for trace, metric, and log/event readbacks. It freezes refs-only fields such as `stage_run_id`, `attempt_id`, `domain_id`, `owner_id`, `route_ref`, `receipt_ref`, `typed_blocker_ref`, `workflow_id`, `task_queue`, `generation`, and `source_fingerprint`; it cannot create a private ledger UI, store payload bodies, write domain truth, create owner receipts, create typed blockers, or claim readiness.
- `cli-command-registry.json`: canonical metadata registry for protected CLI parser adapters, options, output schema refs, and authority boundaries. Command specs bind this metadata instead of repeating it inline; the registry does not execute commands or claim readiness.

### Golden Path

- `golden-path-profile.schema.json`: one ordinary Foundry Agent route plus explicit proof/diagnostic/cleanup/replay/debug variants.
- `default-surface-budget.schema.json`: default / diagnostic / audit / production / cleanup visibility and promotion gate with false authority flags.
- `guardrail-tier-policy.json`: launch-hard, runtime-enforced, domain/human, and audit-only guardrail tier boundary.
- `wrapper-retirement-gate-policy.json`: replacement parity, no-active-caller, no-forbidden-write, and tombstone/provenance static retirement prerequisites, plus the separate domain owner delete/keep/blocker decision shapes.

### Cognitive Computation Kernel

- `cognitive-computation-kernel.json`: Stage-internal strategy kernel for candidate generation, grounded reflection, comparative selection, evolution/revision, meta-review learning, tool affordance boundaries, knowledge binding, and independent quality gates. Tool refs are available affordances with safety/permission/credential/write-scope/forbidden-authority boundaries, not a prescribed workflow. It keeps Route as domain-owner next-step semantics and keeps OPL as refs-only runtime/control-plane owner. Standard Foundry Agents must also declare `contracts/stage_run_canary_evidence.json` as a controlled fixture proving the refs-only closeout shape for candidate, reflection, ranking, revision, meta-review, and independent-gate role artifacts. The fixture can close only as an owner receipt or typed blocker and cannot turn controlled evidence, provider completion, file presence, read-model refresh, or conformance pass into live domain progress. `opl agents conformance` derives an `operator_summary` from this fixture so App/operator surfaces can show the candidate, reflection, comparison, revision, and independent-gate refs; the same gate blocks overclaims such as domain-ready, quality/export-ready, artifact-ready, production-ready, or live-progress fields.

## Reading Rule

- treat this directory as the active OPL framework contract set
- treat `opl framework locate` / `opl_framework_locator` as the stable way for standalone OPL-compatible agents to find their external OPL Framework dependency
- treat Runtime Manager, family runtime attempt, family transition runner, functional agent runtime harness, domain pack compiler, generated interface bundle, and standard domain-agent skeleton contracts as active for the provider-backed family runtime and generated-surface line
- keep domain truth owned by the linked domain repositories, not by this directory
- Foundry Agents should declare and adapt to these framework contracts instead of vendoring or forking their own OPL runtime truth
- treat One Person Lab App as a projection consumer and workbench surface, not as a runtime provider or domain authority
- treat `open_worklist=0`, selected cohorts, verified typed blockers, provider SLO, conformance, descriptor readiness, and refs-only ledger receipts as bounded operator evidence, not as domain ready, App release ready, artifact authority, physical delete authorization, or production ready
- treat standard-agent admission gates as formal inclusion requirements for future candidate domain agents, not as evidence that any current or candidate domain is domain ready or production ready
- treat the domain-handler / provider-hosted stage checkout currentness gate as a pre-admission and pre-launch workspace guard only: it does not write domain truth, sign owner receipts, create typed blockers, or claim runtime/domain readiness
