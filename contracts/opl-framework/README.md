Machine boundary: This document is a human-readable support index for OPL Framework contracts. Machine truth remains in JSON contract files, source, tests, CLI/read-model output, runtime ledgers, and provider/domain receipts.

This directory preserves the active `OPL Framework` runtime and family control-plane contract corpus. `One Person Lab App` and Foundry Agents may consume these contracts, but this directory does not define a second runtime truth for the App or any domain agent.

It is repo-tracked because the current framework needs stable machine-readable inputs for:

- product-layer ownership for `OPL Framework`, `One Person Lab App`, and `Foundry Agents`
- admitted domain-agent / Foundry package catalog projection
- provider-backed runtime attempts
- passive AI-route context transport and currentness readback without a transition table runner or matrix evaluator
- declared stage graph / AI route context support where Codex CLI selects semantic routes and OPL owns only StageRun transport/currentness
- Stage graph / StageRun / isolated Attempt / owner-route-input / ABI-guard boundaries for complex domain agents, where a Stage is one major open semantic judgment and only a progress-terminal decisive Codex Attempt emits the authoritative cross-Stage route
- Codex CLI as the single semantic stage-route owner, with OPL limited to declared-stage transport, identity/currentness checks, attempt durability, and passive projections
- StageRun transport coverage for queue, raw artifact capture, refs-only memory writeback, human gate, retry, dead-letter, and repair requests
- domain pack compiler and generated interface read model that compiles admitted domain packs or standard agent repo contracts into OPL-owned CLI / MCP / Skill / product-entry / OpenAI / AI SDK / sidecar / status / workbench / harness generated-surface handoff projections
- standard domain-agent admission and hosted-action gates that keep candidate identity, domain truth owner, generated default entry, exact managed package source before handler or StageRun launch, provider transport start, Codex runner launch, standard pack ABI, stage artifact contract, execution model, authority boundary, owner receipt, typed blocker, and human gate false-authority requirements machine-readable without admitting any specific domain or claiming readiness

## Target Architecture Contract Groups

These schema files are the machine-readable OPL target architecture surface. Their machine boundary is framework-owned shape, refs, launch / audit / fail-closed semantics, and App/operator projection only. They do not copy domain truth, artifact bodies, memory bodies, owner receipt authority, quality verdicts, production readiness, or App release authority into OPL.

### Target Operating Architecture

- `target-operating-architecture-contract.json`: top-level target architecture contract. It freezes the resource vocabulary, Codex CLI as the only semantic stage-route owner, any-readable-artifact progression, Domain Pack + generated surfaces + authority functions as the standard agent ABI, and OPL as passive transport/currentness/runtime projection only.
- `advisory-knowledge-boundary-contract.json`: family-level boundary for domain-owned Markdown memory. It freezes the default rule that memory refs are `reference_only_prompt_context`, OPL transports body-free refs by default, missing advisory memory does not block launch, and only source/data authority, owner identity, forbidden writes, irreversible mutation, hard reviewer/publication/final export/submission claims, or owner-receipt/typed-blocker claims become hard gates. It also defines `gate_intent` and the Console/App three-column operator projection: reference suggestions, soft gaps, and hard owner gates.
- `okf-context-bundle-contract.json`: OPL-owned OKF context bundle boundary for Google OKF v0.1-compatible Markdown/YAML/crosslink directories. It is consumed by `src/okf-context-bundle.ts` and `opl okf validate|inspect|project-pack|project-repo|native-frontmatter inspect --json` as an advisory, body-free context interchange layer over Foundry Agent pack refs and memory locator refs. `project-repo` reads a domain repo pack compiler input plus optional memory descriptor and writes a single OKF bundle without touching domain truth. Native OKF frontmatter is only an opt-in advisory migration lane for stable domain-owned `agent/**/*.md`; `native-frontmatter inspect` only reports ready/advisory-gap metadata and does not mutate domain files. It is not part of the framework startup required-contract set and does not grant runtime scheduling, domain truth, memory body, owner receipt, typed blocker, quality/export verdict, artifact authority, domain-ready, App-release-ready, or production-ready authority.

### Owner Delta Kernel

- `current-owner-delta.schema.json`: compact default owner / delta / hard-gate / action payload and ordinary next-action root.
- `owner-answer.schema.json`: unified return shape for owner receipt, typed blocker, human decision, and route-back.

### Stage-Internal Quality Cycle

- `stage-quality-cycle-contract.json` defines the non-model Temporal `StageRunWorkflow` parent controller and the bounded `producer | reviewer | repairer | re_reviewer` child Attempt roles. Repeated generation, review, and repair toward the same goal are Attempts; a changed goal, owner, quality gate, or delivery relationship requires a new Stage. The controller owns only child Attempt topology inside one StageRun; the decisive Codex Attempt remains the cross-Stage semantic route owner. OPL validates terminal-role eligibility, route shape, legacy-field absence, finding closure, and declared target identity before the controller materializes the accepted transition, but never judges whether an ABI-valid route is the right domain decision. A missing or rejected decision may fall back only to the action's ordered `required_stage_refs`, or to the current Stage's single `next_stage_refs` successor when no action route exists; manifest file order never resolves a branch.
- `stage-quality-cycle.schema.json` validates per-Stage domain policy without enumerating runtime Attempt instances or granting Stage topology and transition authority to an Attempt overlay.
- `official-knowledge-deliverable-quality-profile.json` requires an explicit quality policy for every AI Stage in MAS, MAG, RCA, OMA, and OBF and one independent top-level Meta Review. Formal Stage Review is risk-selected: a Packaging/Handoff Stage that creates professionally reviewable delivery bytes, freezes canonical bytes, or issues a quality/export/publication/ready claim requires a fresh Review. Immutable refs-only transport or deterministic repackaging of already reviewed bytes may remain primary-only when downstream owner acceptance is retained. Other OPL Agents are not forced to adopt Meta Review.
- Formal Review always uses a fresh child workflow and provider session. `in_thread_refinement` and `protocol_closeout_resume` are non-review operations. Protocol closeout resume runs in a read-only sandbox, is accepted only after binding the returned packet to the existing Attempt, and cannot consume a quality round. Three repair rounds are separate from provider, activity, structured-output, and runtime retry budgets. A domain that declares a Stage quality cycle must distinguish same-Stage repair from a pre-exhaustion cross-Stage route-back in both its machine policy and its reviewer/re-reviewer prompt fragments; every policy-referenced Markdown prompt that uses the machine outcome token `repair_required` must name both branches. `opl agents conformance` also rejects the retired single-branch `repair_required_with_budget_remaining_route_output`, the scope-ambiguous `producer_or_repairer_may_return_terminal_route_decision`, invalid/out-of-repo refs, and missing or reversed scoped route authority. The primary-only producer remains decisive, the formal-Review producer remains non-decisive, and the repairer is never decisive. An exact-ref/hash no-output or failure diagnostic is a consumable progress artifact; literal zero means neither a candidate nor such a diagnostic exists. A draft policy file without a Stage Manifest declaration is not active and remains unaffected.
- StageRun creation is pack-bound: `opl family-runtime attempt create` resolves the compiled Stage manifest and quality policy into `opl_pack_bound_stage_quality_runtime_binding`. `stage_run_id` derives only from domain + stage + durable invocation; the manifest SHA and other immutable launch inputs participate in `stage_run_spec_sha256`. Raw `family-runtime stage-run start` is retired; `stage-run` exposes query only.
- Before formal Review, OPL transport verifies local artifact bytes and creates a content-addressed `opl_transport_artifact_identity_receipt`; external artifacts require an independently readable domain-owned identity receipt. A formal review receipt binds those exact artifact refs/hashes, fresh producer/reviewer sessions, the rubric, finding lineage, and the controller-materialized verdict; file presence or provider completion cannot synthesize the receipt.

## Reading Rule

- treat this directory as the active OPL framework contract set
- treat `opl framework locate` / `opl_framework_locator` as the stable way for standalone OPL-compatible agents to find their external OPL Framework dependency
- treat Runtime Manager, family runtime attempt, domain pack compiler, generated interface bundle, and standard domain-agent skeleton contracts as active for the provider-backed family runtime and generated-surface line
- keep domain truth owned by the linked domain repositories, not by this directory
- Foundry Agents should declare and adapt to these framework contracts instead of vendoring or forking their own OPL runtime truth
- treat One Person Lab App as a projection consumer and workbench surface, not as a runtime provider or domain authority
- treat `open_worklist=0`, selected cohorts, verified typed blockers, provider SLO, conformance, descriptor readiness, and refs-only ledger receipts as bounded operator evidence, not as domain ready, App release ready, artifact authority, physical delete authorization, or production ready
- treat standard-agent admission gates as formal inclusion requirements for future candidate domain agents, not as evidence that any current or candidate domain is domain ready or production ready
- treat the Standard Agent package source gate as latest/LKG resolution plus provenance observation: it does not use version/digest/lock/receipt equality to block progress, write domain truth, sign owner receipts, create typed blockers, or claim runtime/domain readiness
